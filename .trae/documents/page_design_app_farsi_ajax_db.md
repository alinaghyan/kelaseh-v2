# طراحی صفحات (Desktop-first)

## Global Styles
- رنگ‌ها: پس‌زمینه `#F7F8FA`، سطح کارت `#FFFFFF`، متن اصلی `#111827`، متن ثانویه `#6B7280`، رنگ تاکید `#2563EB`، خطا `#DC2626`، موفقیت `#16A34A`.
- تایپوگرافی: فونت Vazirmatn (لوکال)، اندازه‌ها 14/16/20/24، فاصله خطوط 1.6.
- دکمه‌ها: Primary پررنگ (accent)، Secondary با border، حالت hover (تیره‌تر)، حالت disabled کم‌رنگ.
- لینک‌ها: زیرخط در hover، رنگ accent.
- الگوهای وضعیت AJAX: Spinner کوچک، Skeleton برای لیست، Toast برای موفقیت/خطا.

## صفحه: ورود و ثبت‌نام
### Layout
- مرکزچین با Flexbox؛ کارت 420px در وسط؛ پس‌زمینه روشن.
- در موبایل: کارت تمام‌عرض با padding کمتر.

### Meta Information
- Title: "ورود | اپ فارسی"
- Description: "ورود یا ثبت‌نام برای استفاده از داشبورد"
- Open Graph: title/description مطابق بالا.

### Page Structure
1) Header ساده (لوگو/نام اپ)
2) Auth Card (تب‌های ورود/ثبت‌نام)
3) لینک «فراموشی رمز» و متن قوانین (حداقلی)

### Sections & Components
- تب‌ها: «ورود»، «ثبت‌نام».
- فرم ورود: ایمیل، رمز، دکمه ورود، پیام خطا زیر فیلدها.
- فرم ثبت‌نام: ایمیل، رمز، تایید رمز (اختیاری)، دکمه ثبت‌نام.
- حالت‌ها: Loading روی دکمه، پیام موفقیت پس از ارسال ایمیل تایید (اگر فعال باشد).

## صفحه: داشبورد (/app)
### Layout
- دسکتاپ: Layout دو ستونه (Sidebar ثابت 240px + محتوا). Grid/Flex ترکیبی.
- تبلت/موبایل: Sidebar به Drawer (دکمه همبرگری)، محتوا stacked.

### Meta Information
- Title: "داشبورد"
- Description: "مدیریت پروفایل و داده‌ها"

### Page Structure
1) Top Bar: عنوان صفحه، دکمه خروج، نشان نقش
2) Sidebar: لینک‌های «داشبورد»، (اگر مدیر) «پنل مدیر کل»
3) Content: دو کارت اصلی در بالا (پروفایل، ایجاد/ویرایش) + بخش لیست

### Sections & Components
- کارت پروفایل: نام نمایشی، ایمیل (فقط خواندنی)، نقش، دکمه ذخیره.
- فرم داده (Item Form): فیلد عنوان، توضیح، دکمه «ثبت/به‌روزرسانی»، دکمه «انصراف» هنگام ویرایش.
- نوار ابزار لیست: جستجو (متنی)، دکمه «تازه‌سازی»، شمارنده نتایج.
- جدول/لیست داده‌ها: ستون‌ها (عنوان، تاریخ، عملیات)، اکشن‌ها (ویرایش/حذف).
- مودال حذف: تایید حذف + نمایش عنوان آیتم.
- وضعیت‌ها: Empty state (متن + CTA ساخت آیتم)، Error state با دکمه تلاش مجدد.

## صفحه: پنل مدیر کل (/admin)
### Layout
- دسکتاپ: Tabs در بالای محتوا (کاربران/داده‌ها/گزارش) + جدول‌های عریض.
- موبایل: Tabs به Dropdown؛ جدول‌ها به کارت‌های لیستی.

### Meta Information
- Title: "پنل مدیر کل"
- Description: "مدیریت کاربران و نظارت بر داده‌ها"

### Page Structure
1) Header همان داشبورد (هماهنگ)
2) Tab Bar: «کاربران»، «همه داده‌ها»، «گزارش فعالیت»
3) Panel Content

### Sections & Components
- تب کاربران:
  - جدول کاربران: ایمیل، نقش، وضعیت فعال، عملیات.
  - عملیات: تغییر نقش (Select)، فعال/غیرفعال (Toggle)، ذخیره تغییرات.
- تب همه داده‌ها:
  - فیلتر: بر اساس ایمیل مالک/عنوان.
  - جدول داده‌ها: مالک، عنوان، تاریخ، عملیات (ویرایش/حذف).
- تب گزارش:
  - جدول audit: actor، action، entity، زمان.
- کنترل دسترسی UI: اگر نقش مدیر نباشد، ریدایرکت به /app.

## یادداشت پیاده‌سازی UI
- تکنولوژی UI: Bootstrap RTL + jQuery (همه فایل‌ها به صورت لوکال).
- ساختار صفحات: یک `index.php` که با AJAX وضعیت نشست را می‌گیرد و همان‌جا UI را سوئیچ می‌کند.
